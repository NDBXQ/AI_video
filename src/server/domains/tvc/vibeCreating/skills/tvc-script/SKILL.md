# TVC 剧本创作

## 兼容性约束（最高优先级以系统提示词为准）

- 本技能只定义内容结构与字段建议；输出标签白名单与禁止项以系统提示词为准
- 示例仅用于说明结构，不要输出代码围栏

## 核心职责

基于用户产品图和需求分析信息，创作专业的TVC广告剧本大纲。

## 输入参数

- **产品图**：从阶段0获取的产品图片
- **品牌定位**：从阶段0获取的品牌认知和愿景
- **目标客户**：从阶段0获取的目标客户信息和特征
- **美学理念**：从阶段0获取的视觉风格偏好
- **品牌使命**：从阶段0获取的品牌使命
- **核心信息**：从阶段0获取的核心信息
- **广告目的**：从阶段0获取的广告明确目的
- **时长要求**：TVC 时长（秒）
- **目标受众群体**：从阶段0获取的明确目标受众

## 执行步骤

### 1. 产品分析

分析产品图，确定：

- 产品外观特征（颜色、形状、设计风格）
- 产品功能特性（核心卖点）
- 产品使用场景（适用人群、使用环境）
- 产品情感属性（高端、亲民、科技感等）

### 2. 品牌定位结合

结合阶段0的需求分析：

- 将品牌愿景融入剧本主题
- 根据目标受众确定故事风格
- 根据美学理念确定视觉风格
- 根据品牌使命确定情感基调
- 根据核心信息确定传播重点
- 根据广告目的确定剧本结构

### 3. 剧本创作

基于产品分析和需求分析，创作剧本大纲，包含：

- 广告主题（符合品牌愿景）
- 创意概念（针对目标受众）
- 故事线概述（体现品牌使命）
- 核心传播点（传递核心信息）
- 情感基调（符合美学理念）
- 产品展示方式（自然融入剧情）

### 4. 目的导向

根据广告目的优化剧本：

- **吸引客户留资**：强化产品价值，引导行动
- **建立品牌形象**：强调品牌调性，传递情感
- **实现产品销售**：突出产品卖点，激发购买欲
- **提升品牌认知**：创造记忆点，增加曝光

## 输出内容结构

### 输出格式（阶段1专用）

你必须以 `<script>` + 纯 Markdown 的形式输出两部分内容：

1) `<script>...</script>`：`<script>` 标签内只写 Markdown（这是“剧本 markdown”，用于独立展示与入库）
2) `</script>` 之后：只写纯 Markdown（这是“对话框展示内容”）

### `<script>` 内的“剧本 markdown”要求

- 只输出 Markdown，不要输出任何 XML/HTML 结构化标签
- 推荐包含：标题、一句话核心主张、创意概念、故事线/节奏、关键画面与产品露出、口播/字幕建议
- 内容必须可直接给人阅读与评审，避免占位符
- 固定格式（强制，便于前端解析与结构化展示）：
  - 必须使用 1 个 H1 标题：`# {标题}`（仅 1 行）
  - 必须使用多个 H2 小节：`## {小节标题}`（不要使用 `###` 及更深层级）
  - 推荐按如下小节标题输出（标题可微调，但必须保留“故事线/口播”两节，且小节顺序尽量一致）：
    - `## 一句话核心主张`
    - `## 创意概念`
    - `## 故事线/节奏`
    - `## 关键画面与产品露出`
    - `## 口播/字幕`
    - `## 约束与备注`（可选）
  - “故事线/节奏”小节必须使用逐行时间轴格式（每行一条，至少 4 行）：
    - 格式：`{起始}-{结束}s：{画面/动作/卖点/字幕}`（例如：`3-10s：产品登场，字幕突出核心卖点`）
    - 只允许使用半角冒号 `:` 或中文冒号 `：` 作为分隔符
  - “口播/字幕”小节必须使用逐行口播格式（每行一条，至少 2 行）：
    - 格式：`（{起始}-{结束}s）{角色}：{台词/字幕}`（例如：`（10-15s）旁白：一句记忆点口播`）
    - 角色建议：旁白 / 主播 / 用户 / 店员 / KOL 等
  - 避免：表格、代码围栏、JSON、引用块、内嵌 HTML（会影响解析与展示一致性）

### `</script>` 之后的“对话框 Markdown”要求

- 必须包含三行阶段导航（各自独立成行）：
  - 当前阶段：1 剧本
  - 下一步：…
  - 关键问题：…（信息充足则写“关键问题：无”）
- 简洁说明你产出的剧本方向，并引导用户确认或提出修改点

## 注意事项

- 剧本要符合TVC时长要求（通常15-60秒）
- 产品展示要自然融入剧情，避免硬广感
- 核心传播点要清晰明确
- 情感基调要与品牌定位一致
- 必须基于阶段0收集的需求分析信息创作
- 剧本要服务于明确的广告目的
- 必须输出一个完整闭合的 `<script>...</script>`，并确保 `<script>` 内只包含 Markdown
- 生成前自检：是否只使用允许标签（仅 `<script>`）；是否避免输出代码围栏/JSON；`<script>` 是否完整闭合
